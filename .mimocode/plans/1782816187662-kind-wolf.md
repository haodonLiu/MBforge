# Code Review Fixes — OpenKB Migration

> **Date**: 2026-06-30

## Fix 1 (High): Simplify async bridging in `adapter.py`

**File**: `src/mbforge/openkb/adapter.py:56-75`

**Problem**: `search()` has complex async/sync bridging with `concurrent.futures.ThreadPoolExecutor` + nested `asyncio.run()`. The running-loop detection branch is dead code (always called from sync context) and would be problematic if hit.

**Fix**: Always use `asyncio.run()`:

```python
def search(self, query: str, top_k: int = 10) -> dict[str, Any]:
    from .query import search_wiki
    return asyncio.run(search_wiki(query, str(self._wiki_dir), top_k))
```

Remove the `concurrent.futures` import and running-loop detection.

---

## Fix 2 (High): Simplify async bridging in `runner.py`

**File**: `src/mbforge/pipeline/runner.py:107-116`

**Problem**: `compile_wiki` runs via `concurrent.futures.ThreadPoolExecutor` + `asyncio.run()`, creating a thread-within-a-thread. Unnecessary complexity.

**Fix**: Since `run_pipeline()` is always called from `asyncio.to_thread()` (sync context), use `asyncio.run()` directly:

```python
asyncio.run(adapter.compile_wiki(doc_name, doc_id, extracted.page_count))
```

Remove the `concurrent.futures` import and running-loop detection block.

## Fix 3 (Medium): Add debug logging in `query.py`

**File**: `src/mbforge/openkb/query.py:104-106`

**Problem**: `except Exception: continue` silently swallows file read errors.

**Fix**: Add logger.debug:

```python
except Exception as e:
    logger.debug("Failed to read %s: %s", f, e)
    continue
```

## Fix 4 (Medium): Move `json` import to top in `knowledge_base.py`

**File**: `src/mbforge/core/knowledge_base.py:103-112`

**Problem**: `import json` inside `get_document_tree()` function body. Minor inefficiency on hot path.

**Fix**: Add `import json` at the top of the file with other stdlib imports.

## Fix 5 (Medium): Add config migration log

**File**: `src/mbforge/utils/config.py:68-76`

**Problem**: Existing configs with `embed`/`rerank` fields are silently ignored. Users may not notice.

**Fix**: In `load_global_config()`, after successful `model_validate()`, log a warning if old fields are present:

```python
data = load_json(_CONFIG_PATH)
if data is not None:
    if "embed" in data or "rerank" in data:
        logger.info("Legacy embed/rerank config fields ignored (removed in OpenKB migration)")
    try:
        return AppConfig.model_validate(data)
    except Exception:
        pass
```
