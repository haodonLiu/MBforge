# Plan: 后端路由结构整理与可维护性提升

## Context

经过 Settings 审查和前后端路由对齐修复后，后端 routers/ 目录从 12 个文件增长到 18 个。存在以下结构问题：

1. **environment.py 过于臃肿**（15 条路由）— 混杂了 health 检查、sidecar 管理、资源下载/删除/测试，职责不清
2. **text.py 路由挂载路径不一致** — 挂载在 `/api/v1`，但内部路由混杂 `/text/chunk`、`/classify/*`、`/extract/*` 三种前缀
3. **settings.py 有死代码** — POST handler 与 GET 行为完全相同，是冗余兼容层
4. **audit.py 零消费者** — 前端无任何组件调用，属于死代码
5. **server.py 挂载双前缀** — 路由定义带 `/api/v1` 前缀，但挂载在 `/api/v1/models`，导致实际路径变为 `/api/v1/models/api/v1/...`
6. **documents.py 导入顺序** — `import asyncio` 在相对导入之后

## Changes

### 1. 拆分 environment.py 为三个聚焦路由器

**原文件** `src/mbforge/routers/environment.py`（15 routes）拆为：

| 新文件 | 前缀 | 路由 |
|--------|------|------|
| `health.py` | `/api/v1` | health, sidecar/status, sidecar/restart, environment/check |
| `resource.py` | `/api/v1` | resource/download, resource/cache-dir-info, resource/delete, resource/delete-subfile, resource/download-subfile, resource/test, resources/check, resources/status, resources/model-path, resources/catalog, resources/refresh-paths |

**删除** `environment.py`。

### 2. 修复 text.py 路由挂载路径

**文件** `src/mbforge/routers/text.py`

将路由路径改为与前端一致的完整路径：
- `/chunk` → `/text/chunk`（已有）
- `/classify/page` → `/classify/page`（已有）
- `/extract/esmiles-candidates` → `/extract/esmiles-candidates`（已有）

text.py 保持挂载在 `/api/v1`，路由路径已正确，无需修改。

### 3. 清理 settings.py 死代码

**文件** `src/mbforge/routers/settings.py`

删除 `settings_post` handler（POST /api/v1/settings）— 它与 GET 行为完全相同，前端只用 GET 和 PUT。

### 4. 删除 audit.py

**删除** `src/mbforge/routers/audit.py`

前端 `audit.ts` 零生产消费者，仅自身测试文件引用。后端 audit 路由为 stub。

### 5. 修复 app.py 路由注册

**文件** `src/mbforge/app.py`

- 替换 `environment.router` 为 `health.router` + `resource.router`
- 移除 `audit.router` 注册
- 更新 tags

### 6. 修复 documents.py 导入顺序

**文件** `src/mbforge/routers/documents.py`

将 `import asyncio` 移到相对导入之前。

### 7. 清理 frontend 死代码

**文件** `frontend/src/api/tauri/audit.ts`

删除此文件及其 barrel export。
