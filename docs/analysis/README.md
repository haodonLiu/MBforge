## 🎯 如何使用这些报告

### 场景 1: 技术评审会议
1. **会前**: 团队成员阅读 [精简总结](./defects-summary-zh.md)（15 分钟）
2. **会中**: 技术负责人展示 [完整报告](./project-defects-analysis-2026-07-09.md) 第一、二、六节（30 分钟）
3. **会后**: 分配任务，开发者参考 [行动计划](./critical-issues-action-plan.md) 执行

### 场景 2: 向管理层汇报
1. 提交 [精简总结](./defects-summary-zh.md) 的"核心问题速览"和"关键指标目标"两节
2. 强调：
   - **风险**: 3 个月后用户流失 + 回归 bug 频发
   - **成本**: 第 1 周 10 人时，第 1 月 50 人时
   - **收益**: 用户体验改善 + 代码质量达标 + 可快速迭代新功能

### 场景 3: 新人 onboarding
1. 新工程师阅读 [完整报告](./project-defects-analysis-2026-07-09.md) 了解项目历史和技术债
2. 分配 1-2 个 P2/P3 问题作为熟悉代码库的练习
3. 参与 P1 问题修复，学习最佳实践
