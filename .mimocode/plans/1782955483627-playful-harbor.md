# MBForge Dear PyGui 原生前端重写计划

## 目标

用 Dear PyGui (dpg) 完全重写 MBForge 前端，替代 React+pywebview 方案，实现纯 Python 原生桌面应用。

## 核心模块设计

### 1. 入口与启动 (`entry.py`, `__init__.py`)

替换当前 pywebview 方案：

```python
# entry.py
import dearpygui.dearpygui as dpg

def launch(port: int = 18792, dev: bool = False):
    """启动 MBForge 原生 GUI"""
    dpg.create_context()
    dpg.create_viewport(title="MBForge", width=1400, height=900)
    
    # 初始化主题
    setup_themes()
    
    # 创建主窗口
    app = MBForgeApp(port=port, dev=dev)
    app.create()
    
    dpg.setup_dearpygui()
    dpg.show_viewport()
    dpg.start_dearpygui()
    dpg.destroy_context()
```

`__main__.py` 修改：
- 移除 pywebview 依赖
- `--gui` 标志调用 `entry.launch()`
- 后台启动 FastAPI 后端（复用现有逻辑）

## 关键技术点

### 1. 线程安全

Dear PyGui 是单线程的，API 调用需要在后台线程执行：

```python
import threading

def api_call_with_loading(api_func, callback, *args):
    """带加载状态的 API 调用"""
    dpg.configure_item("loading_spinner", show=True)
    
    def _worker():
        try:
            result = api_func(*args)
            # 使用 dpg 触发 UI 更新
            dpg.set_value("result_tag", result)
        except Exception as e:
            show_toast(f"Error: {e}", level="error")
        finally:
            dpg.configure_item("loading_spinner", show=False)
    
    threading.Thread(target=_worker, daemon=True).start()
```

### 2. SSE 流式处理

```python
def stream_sse(url: str, params: dict, on_chunk: Callable):
    """在后台线程处理 SSE 流"""
    def _worker():
        with httpx.stream("GET", url, params=params) as response:
            for line in response.iter_lines():
                if line.startswith("data: "):
                    data = json.loads(line[6:])
                    if data.get("event") == "chunk":
                        on_chunk(data.get("delta", ""))
                    elif data.get("event") == "done":
                        break
    
    threading.Thread(target=_worker, daemon=True).start()
```

## 风险与缓解

### 风险 1: Dear PyGui 组件能力有限
**缓解**：
- 使用 `child_window` 实现可滚动区域
- 使用 `table` 实现数据表格
- 使用 `tree_node` 实现文件树
- 必要时使用 `draw_list` 自定义绘制

### 风险 2: 线程安全问题
**缓解**：
- 所有 API 调用在后台线程执行
- UI 更新通过 `dpg.set_value()` 或回调
- 使用 `threading.Lock` 保护共享状态

### 风险 3: SSE 流式处理复杂
**缓解**：
- 使用 `httpx.stream()` 处理 SSE
- 后台线程接收事件，主线程更新 UI
- 实现重连机制

### 风险 4: 开发周期长
**缓解**：
- 分阶段实施，每阶段交付可用版本
- 优先实现核心功能（欢迎页、工作区）
- 高级功能（SAR 分析）可后续迭代

## 总结

本计划将 MBForge 前端从 React+pywebview 完全重写为 Dear PyGui 原生应用。

**优势**：
- 纯 Python 实现，无需 Node.js/React 构建
- GPU 加速渲染，性能优异
- 原生外观，更好的系统集成
- 更小的部署包（无需前端构建产物）

**工作量**：约 10 个工作日

**交付物**：
- 完整的 Python 原生 GUI 应用
- 7 个页面全部实现
- 暗色/亮色主题
- 中英文国际化
- SSE 流式响应支持
