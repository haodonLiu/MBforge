# 前端可维护性提升计划

## 问题总结

| # | 问题 | 严重性 | 影响文件数 |
|---|------|--------|-----------|
| 1 | Import 路径不统一（3 种风格混用，含 6 处错误路径） | 高 | ~90+ |
| 2 | 重复类型定义（MoleculeRecord_） | 低 | 2 |
| 3 | 超标组件未拆分（5 个 >300 行） | 高 | 5 |

**执行顺序**：Phase 2 → Phase 1 → Phase 3

---

## Phase 2：重复类型清理（5 分钟）

**涉及文件**：
- `frontend/src/types/index.ts` — `MoleculeRecord` 添加 `smiles?: string`
- `frontend/src/api/tauri/molecule.ts` — 删除 `MoleculeRecord_`，改用 `import { MoleculeRecord } from '@/types'`

---

## Phase 1：Import 路径统一

**目标**：统一使用 `@/` 别名，消除相对路径超 2 级和错误路径。

### 1.1 修复 Welcome.tsx 错误路径（6 处）

```typescript
// 错误: import Button from '../components/ui/Button'
// 正确: import Button from '@/components/ui/Button'
```

### 1.2 批量替换相对路径

按子目录逐批处理：
1. `components/` 根级（~15 个文件）
2. `components/molecule/`（~15 个文件）
3. `components/project/`（~10 个文件）
4. `components/settings/`（~15 个文件）
5. `components/sar/`（~10 个文件）
6. `components/welcome/`, `components/notes/`, `components/discover/` 等

替换规则：
- `../ui/Button` → `@/components/ui/Button`
- `../../types` → `@/types`
- `../../../types` → `@/types`
- `../../hooks/xxx` → `@/hooks/xxx`
- `../../api/tauri/xxx` → `@/api/tauri/xxx`
- `../../context/xxx` → `@/context/xxx`
- `../icons` → `@/components/icons`
- `../animations/xxx` → `@/components/animations/xxx`
- `../../styles/xxx` → `@/styles/xxx`

**例外**：`components/ui/` 内部互相引用保留相对路径（如 `./Badge`）
