# MBForge Pure Python Rewrite Plan

## Goal

Replace ALL Rust/Tauri code with pure Python, running as a **web application** (FastAPI backend + browser-accessible React frontend). Work directly on `main` branch. Rust backup archived at `../mbforge_rust/`. Keep all existing Python code (backends, parsers, utils). Use mature libraries (LangChain/LangGraph, pdfplumber) instead of reinventing.

## Key Design Decisions

1. **Skip LlamaIndex** ‚Äî PyMuPDF + pdfplumber + existing RapidOCR already cover PDF extraction. LlamaIndex adds 200+ transitive deps for marginal value. Use pdfplumber for structured text/tables, PyMuPDF for rendering.
2. **Consolidate 147 Rust IPC commands ‚Üí 65 REST endpoints** ‚Äî merge related commands (e.g. `mol_store_*` + `mol_admin_*` ‚Üí `/api/v1/molecule/*`).
3. **SSE for all streaming** ‚Äî Agent chat, KB search, ingest progress all use Server-Sent Events via `sse-starlette`.
4. **Frontend migration is surgical** ‚Äî Only `frontend/src/api/tauri/*.ts` files change (`invoke` ‚Üí `fetch`). All React components stay untouched since they call wrapper functions.
5. **Agent uses LangGraph `create_react_agent`** ‚Äî Pre-built ReAct pattern with tool dispatch, streaming, memory. Supports OpenAI/Anthropic/Ollama via LangChain chat model abstractions.

## Document Pipeline (`pipeline/`)

### `pipeline/runner.py`
Orchestrates 5 stages:
1. **Classify** (`classify.py`): Determine doc type (paper/patent/report) from content heuristics
2. **Extract** (`extract_text.py`): pdfplumber for text+tables, PyMuPDF for images, RapidOCR for scanned pages
3. **Segment** (`segment.py`): Heading detection, section splitting
4. **Chunk** (`chunk.py`): Sentence-aware chunking (configurable size/overlap, default 512/128)
5. **Index** (`index.py`): Embed chunks via qwen3 ‚Üí index into Zvec

## Frontend Migration Strategy

### Approach: Surgical replacement of `api/tauri/*.ts` wrappers

1. **Create `frontend/src/api/http-client.ts`**: Base HTTP client with error handling
2. **Modify each `api/tauri/*.ts`**: Replace `invoke<T>('cmd', params)` with `fetch('/api/v1/...')`
3. **Create `frontend/src/api/sse-client.ts`**: SSE client for streaming events
4. **Replace Tauri event listeners**: `listen('event-name')` ‚Üí `EventSource('/api/v1/events')`

### Mapping (example):

```typescript
// Before (Tauri)
export async function openProject(root: string): Promise<ProjectResponse> {
  return invokeWithError('open_project', { root });
}

// After (HTTP)
export async function openProject(root: string): Promise<ProjectResponse> {
  return httpPost('/api/v1/project/open', { root });
}
```

### Events migration:
```typescript
// Before (Tauri)
listen('agent-stream-chunk', (event) => { ... });

// After (SSE)
const es = new EventSource('/api/v1/agent/session/abc/chat/stream');
es.addEventListener('chunk', (event) => { ... });
```

- FastAPI serves `frontend/dist/` as static files in production
