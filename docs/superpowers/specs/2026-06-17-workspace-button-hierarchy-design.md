# 工作区按钮层级重设计

## 背景

MBForge 工作区页面（`Workspace → ProjectView → ProjectDashboard`）顶部当前排列了四个按钮：

- **扫描**
- **索引**
- **分子扫描**
- **设置**

用户反馈这三个操作按钮存在明显重叠，且层级不清晰：

1. **流程重叠**：`handleIndex` 内部已经调用了 `scanProjectFiles()`，再单独放置「扫描」按钮会让用户重复触发同一段流程。
2. **入口重叠**：ProjectScope 有「导入 PDF」、DocumentList 有「立即处理」，顶部按钮与这些入口在概念上互相干扰。
3. **概念重叠**：普通用户难以区分「扫描」「索引」「分子扫描」三者的执行顺序和产出结果。
4. **层级错位**：「设置」是全局配置入口，不应与当前项目的操作按钮平级排列。

## 目标

- 让工作区顶部的按钮只保留**项目级、高频、语义清晰**的操作。
- 合并扫描与索引为一个「同步文献」主操作。
- 将「分子扫描」改名为更直白的「检测分子结构」，作为次级项目操作。
- 弱化设置入口，使其不再占用主操作区域。
- 保持现有进度条、统计卡片、错误提示等反馈机制。

## 设计原则

1. **单一入口原则**：同一个流程不要让用户在多个位置以不同名称触发。
2. **动词优先**：按钮文案使用用户能理解的动词（同步、检测），避免技术术语（扫描、索引）。
3. **项目级 vs 文档级分离**：全局项目操作放在顶部，单个文档操作保留在 DocumentList 行内。
4. **配置与操作分离**：设置类入口不与应用主操作并列。

## 最终方案

### 顶部按钮层级

```
[ 同步文献 ]  [ 检测分子结构 ]         ⚙
```

- 左侧两个为项目级主操作。
- 右侧仅用设置图标，与 Sidebar 底部的设置入口保持一致。

### 按钮 1：同步文献

**位置**：ProjectDashboard header 左侧主按钮。

**触发逻辑**：

1. 调用 `scanProjectFiles(projectRoot)` 扫描 `papers/` 与 `notes/` 目录。
2. 扫描完成后，自动调用 `indexProjectRust(projectRoot)` 建立向量索引。
3. 同时调用 `enqueueUnresolvedDocuments(projectRoot)` 将未处理文档加入处理队列。
4. 完成后刷新文档列表与统计卡片。

**状态文案**：

| 阶段 | 按钮文案 |
|------|----------|
| 空闲 | `同步文献` |
| 扫描中 | `扫描文件中…` |
| 索引中 | `建立索引中…` |

**错误处理**：

- 未找到文件：显示提示，引导用户将 PDF 放入 `papers/`、将 Markdown 笔记放入 `notes/`。
- 索引引擎离线：提示用户检查 Sidecar 或重启应用。
- 操作超时：显示超时错误，允许用户重试。

### 按钮 2：检测分子结构

**位置**：「同步文献」右侧的次级按钮。

**触发逻辑**：

1. 筛选 `docs.filter(d => d.doc_type === 'pdf')`。
2. 调用 `batchQuickMoldetScan(projectRoot, pdfIds)` 批量检测 PDF 中的分子图像。
3. 完成后刷新文档列表与分子统计。

**状态文案**：

| 阶段 | 文案 |
|------|------|
| 空闲 | `检测分子结构` |
| 进行中 | `检测分子中 {{current}}/{{total}}` |
| 完成 | `已扫描 {{scanned}} 个 PDF，{{withMolecules}} 个含分子结构` |

**错误处理**：

- 项目中没有 PDF：禁用按钮并提示。
- 检测失败：显示 toast 错误，保留成功结果。

### 设置入口

- 从 ProjectDashboard header 中移除文字按钮 `设置`。
- 在 header 右上角保留 `SettingsIcon` 图标按钮，点击跳转 `/settings`。
- Sidebar 底部已有设置入口，两处不冲突。

### 统计与进度

保留现有 4 张统计卡片，含义保持不变：

| 卡片 | 数值 | 含义 |
|------|------|------|
| 文献 | 总文档数 | 项目内 PDF + Markdown 总数 |
| 段落 | 已索引段落数 | `indexResult.sections` |
| 已索引 | `indexedCount / totalDocs` | 已建立向量索引的文档比例 |
| 分子 | 总分子数 + 已确认数 | 来自 `molAdminStoreStats` |

同步与检测的进度条保留在文档列表上方，仅将标题文案调整为更友好的表述。

## 需要修改的文件

| 文件 | 改动 |
|------|------|
| `frontend/src/components/ProjectView.tsx` | 合并 `handleScan` 与 `handleIndex` 为 `handleSync`；保留 `handleMoldetScan` 但可重命名 prop。 |
| `frontend/src/components/project/ProjectDashboard.tsx` | 调整 header 按钮：两个主操作 + 设置图标；合并 loading 状态。 |
| `frontend/src/i18n/locales/zh-CN.json` | 新增/调整 `project.sync`、`project.syncing`、`project.detectMolecules` 等文案。 |
| `frontend/src/i18n/locales/en.json` | 同步更新英文文案。 |
| `frontend/src/styles/workspace.css`（可选） | 微调 header 按钮间距与响应式布局。 |

## 边界情况

- 同步过程中用户切换项目：当前操作自然中断，新项目加载后按钮恢复空闲状态。
- 同步与检测同时触发：保持现有互锁逻辑，任一进行中时禁用另一按钮。
- 空项目：「同步文献」仍可点击，但扫描后提示无文件；「检测分子结构」禁用。

## 未采纳方案

### 方案 B：主按钮 + 下拉菜单

在「同步」旁提供「仅扫描」「重新索引」等高级选项。优点是保留细分控制，缺点是增加了层级和概念数量，不符合本次「减少重叠」的核心目标。

### 方案 C：全自动后台任务

把分子检测也合并进同步流程，用户完全不需要手动触发。优点是界面最简，缺点是失去对计算密集型任务的显式控制，且 MoldDet 耗时较长，自动执行可能让用户感到意外。

## 验收标准

- [ ] 工作区顶部仅显示「同步文献」「检测分子结构」两个文字按钮和一个设置图标。
- [ ] 点击「同步文献」后依次完成扫描、索引、enqueue 并刷新列表。
- [ ] 点击「检测分子结构」后仅对 PDF 执行批量 MoldDet 检测。
- [ ] 两个操作互锁，任一进行中时另一按钮禁用。
- [ ] 无文件/无 PDF 等边界情况有明确提示。
- [ ] 中英文文案均已更新。
