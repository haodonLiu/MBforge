# Plan: MBForge CLI (`mbforge` command)

## Goal

Add a standalone CLI binary `mbforge` to MBForge, enabling headless PDF processing, knowledge base search, project management, and molecule operations from the terminal.

**V1 scope**: PDF processing pipeline + KB search + project init + molecule stats. Remaining subcommands tracked in TODO.

## Architecture

New crate `mbforge-cli` in `src-tauri/crates/mbforge-cli/`. Standalone binary, no Tauri WebView. Depends on existing workspace crates for domain logic.

```
mbforge-cli (binary)
├── mbforge-pipeline  (run_pipeline, SourceInput, PipelineContext)
├── mbforge-domain    (Project, KnowledgeBase, MoleculeDatabase)
├── mbforge-infra     (AppConfig, constants, sidecar_client)
└── mbforge-chem      (SMILES validation)
```

**Tauri dependency note**: workspace crates depend on Tauri for `#[tauri::command]` wrappers. The CLI compiles these but never initializes a Tauri app. A follow-up task will add `tauri` feature flags to domain/pipeline crates to make the CLI fully Tauri-free. For V1 this is acceptable since the CLI only calls core domain functions, not Tauri commands.

### 2. `src-tauri/crates/mbforge-cli/src/main.rs`

Entry point: parse args via clap, dispatch to subcommand modules.

```rust
use clap::{Parser, Subcommand};

#[derive(Parser)]
#[command(name = "mbforge", version, about = "MBForge CLI — molecular science knowledge platform")]
struct Cli {
    #[command(subcommand)]
    command: Commands,
}

#[derive(Subcommand)]
enum Commands {
    /// Process a PDF document through the extraction pipeline
    Process {
        /// Path to the PDF file
        path: String,
        /// Project root directory (default: auto-detect)
        #[arg(short, long)]
        project_root: Option<String>,
        /// User request / extraction goal
        #[arg(short, long)]
        request: Option<String>,
    },
    /// Search the knowledge base
    Search {
        /// Search query
        query: String,
        /// Project root directory
        #[arg(short, long)]
        root: String,
        /// Number of results (default: 5)
        #[arg(short, long, default_value = "5")]
        top_k: usize,
    },
    /// Project management
    Project {
        #[command(subcommand)]
        action: ProjectAction,
    },
    /// Molecule database operations
    Molecule {
        #[command(subcommand)]
        action: MoleculeAction,
    },
}
```

### 3. `src-tauri/crates/mbforge-cli/src/process.rs`

PDF processing subcommand. Implements `PipelineReporter` for terminal progress display using `indicatif`.

Key logic:
- Start Python sidecar if needed (via `sidecar_client` or direct spawn)
- Create `SourceInput` and `PipelineContext`
- Implement `CliReporter` struct that implements `PipelineReporter` trait
  - `StageStart` → spinner message
  - `StageProgress` → progress bar message
  - `StageComplete` → green checkmark
  - `StageFailed` → red X + error
  - `StageWarning` → yellow warning
- Call `run_pipeline(input, &ctx).await`
- Shutdown sidecar on completion

### 4. `src-tauri/crates/mbforge-cli/src/search.rs`

Knowledge base search. Uses `get_or_init_kb()` + `kb.search()` directly.

Output: formatted table with score, title, text snippet, page range.

### 5. `src-tauri/crates/mbforge-cli/src/project.rs`

Project init/list operations.
- `init <path>`: calls `Project::bootstrap()` or equivalent
- `list`: reads recent projects from config

### 6. `src-tauri/crates/mbforge-cli/src/molecule.rs`

Molecule stats. Opens `MoleculeDatabase` and prints record count, clusters, labels distribution.

### 7. `src-tauri/crates/mbforge-cli/src/sidecar.rs`

Sidecar lifecycle management for CLI context:
- Detect Python path (same logic as main.rs)
- Spawn sidecar process
- Health check loop
- Graceful shutdown on drop
