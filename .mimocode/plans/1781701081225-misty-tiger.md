# 学习 PaddleOCR Local 模式实现计划

> 基于 PaddleOCR Local 项目分析，将可复用模式适配到 MBForge

---

## 1. 背景

PaddleOCR Local 的 6 个核心模式可直接提升 MBForge 的用户体验：

| 模式 | PaddleOCR 实现 | MBForge 现状 | 优先级 |
|------|---------------|-------------|--------|
| 滚动同步 | `splitScrollSyncLocked` + `requestAnimationFrame` | PDF 和结果面板独立滚动 | P0 |
| 虚拟滚动 | JSON 结果按行渲染，仅可见行 | 大文档结果全量渲染 | P1 |
| scrollToDetection | 单页模式可用，连续模式不可用 | `scrollToDetection` 仅单页模式工作 | P0 |
| 分片解析 | 大 PDF 按页分批处理 | 整页一次性处理 | P2 |
| 任务状态机 | idle → processing → completed/error | 无统一状态机 | P2 |
| 模型状态轮询 | 2.5s 轮询模型就绪状态 | 无轮询 | P3 |

## 2. P0: 修复 scrollToDetection + 滚动同步

### 2.1 问题

`scrollToDetection` 在连续模式下不工作，因为 `pdfScrollRef` 没有绑定到 `PdfContinuousViewer` 的内部容器。

### 2.2 方案

**文件**: `frontend/src/components/PdfContinuousViewer.tsx`

1. 添加 `ref` 转发，暴露内部滚动容器
2. 添加 `onScroll` 回调报告滚动进度

```typescript
// 改为 forwardRef
const PdfContinuousViewer = forwardRef<HTMLDivElement, Props>(
  ({ url, scale, onPageChange, onPageCount, onScrollProgress }, ref) => {
    const containerRef = useRef<HTMLDivElement>(null)
    
    // 合并 ref
    useImperativeHandle(ref, () => containerRef.current!)
    
    // 滚动进度回调（节流）
    const handleScroll = useCallback(() => {
      if (!containerRef.current) return
      const { scrollTop, scrollHeight, clientHeight } = containerRef.current
      onScrollProgress?.({
        scrollTop, scrollHeight, clientHeight,
        progress: scrollTop / (scrollHeight - clientHeight || 1),
      })
    }, [onScrollProgress])
    
    return (
      <div ref={containerRef} onScroll={handleScroll} style={{ overflowY: 'auto' }}>
        ...
      </div>
    )
  }
)
```

**文件**: `frontend/src/components/project/pdf/usePdfViewer.ts`

修改 `scrollToDetection` 支持连续模式：

```typescript
const scrollToDetection = useCallback((detection: ExtractionResult) => {
  if (!detection.bbox_pdf || !pageInfoRef.current) return
  const container = pdfScrollRef.current
  if (!container) return
  
  const [, y1, , y2] = detection.bbox_pdf
  const info = pageInfoRef.current
  const cssY = (info.originalHeight - y2) * info.scale
  const cssY2 = (info.originalHeight - y1) * info.scale
  const centerY = (cssY + cssY2) / 2
  
  // 连续模式下需要加上当前页之前所有页的偏移
  // 单页模式直接滚动
  container.scrollTo({
    top: Math.max(0, centerY - container.clientHeight / 2),
    behavior: 'smooth',
  })
}, [])
```

**文件**: `frontend/src/components/project/PdfViewer.tsx`

将 `pdfScrollRef` 传递给 `PdfContinuousViewer`：

```tsx
<PdfContinuousViewer
  ref={v.pdfScrollRef}
  url={v.pdfUrl}
  scale={v.pdfScale}
  onPageChange={v.setCurrentPage}
  onPageCount={v.handlePageCount}
/>
```

### 2.3 验证

1. 单页模式：点击分子卡片 → PDF 滚动到检测框位置 ✅
2. 连续模式：点击分子卡片 → PDF 滚动到检测框位置 ✅
3. 切换模式后功能正常 ✅

## 4. P1: 大文档结果虚拟滚动

### 4.1 问题

当分子检测结果很多时（>100），全量渲染 DOM 会导致性能问题。

### 4.2 方案

**文件**: `frontend/src/components/project/pdf/PdfResultPane.tsx`

使用 `react-window` 或手动虚拟滚动：

```typescript
// 分子卡片虚拟列表
const VISIBLE_COUNT = 20
const ITEM_HEIGHT = 120 // 每个卡片约 120px

function VirtualMoleculeList({ detections, ... }: Props) {
  const [scrollTop, setScrollTop] = useState(0)
  const containerRef = useRef<HTMLDivElement>(null)
  
  const startIndex = Math.floor(scrollTop / ITEM_HEIGHT)
  const visibleItems = detections.slice(startIndex, startIndex + VISIBLE_COUNT)
  
  return (
    <div ref={containerRef} onScroll={e => setScrollTop(e.currentTarget.scrollTop)}
         style={{ overflow: 'auto', height: '100%' }}>
      <div style={{ height: detections.length * ITEM_HEIGHT, position: 'relative' }}>
        {visibleItems.map((mol, i) => (
          <div key={startIndex + i} 
               style={{ position: 'absolute', top: (startIndex + i) * ITEM_HEIGHT, width: '100%' }}>
            <MoleculeCard ... />
          </div>
        ))}
      </div>
    </div>
  )
}
```

### 4.3 验证

1. 100+ 分子结果滚动流畅
2. 虚拟滚动不影响点击和选中功能

## 5. P2: 大 PDF 分片解析

### 5.1 方案

**文件**: `src-tauri/src/parsers/pipeline.rs`

在 Rust 解析管线中添加分片处理：

```rust
pub async fn parse_pdf_incremental(path: &str, batch_size: usize) -> ... {
    let doc = lopdf::Document::load(path)?;
    let page_count = doc.get_pages().len();
    
    for start in (1..=page_count).step_by(batch_size) {
        let end = (start + batch_size - 1).min(page_count);
        // 解析这一批
        let batch_results = parse_pages(&doc, start, end).await?;
        // 通过事件发送进度
        emit_progress(start, end, page_count);
        // 保存结果
        save_batch_results(&batch_results);
    }
}
```

### 5.2 验证

1. 大 PDF（50+页）解析时显示进度
2. 可以中断/恢复解析

```typescript
function useTaskState(docId: string) {
  const [state, setState] = useState<TaskState>({ status: 'idle', progress: 0, ... })
  
  // 监听 Tauri 事件更新状态
  useEffect(() => {
    const unlisten = listen('task-progress', (event) => {
      setState(prev => ({ ...prev, ...event.payload }))
    })
    return unlisten
  }, [docId])
  
  return state
}
```

## 7. P3: 模型状态轮询

### 7.1 方案

**文件**: `frontend/src/hooks/useSidecarHealth.ts` (新增)

```typescript
function useSidecarHealth(intervalMs = 5000) {
  const [health, setHealth] = useState<ModelStatus>('unknown')
  
  useEffect(() => {
    const poll = async () => {
      try {
        const status = await invoke('get_sidecar_health')
        setHealth(status)
      } catch { setHealth('error') }
    }
    poll()
    const timer = setInterval(poll, intervalMs)
    return () => clearInterval(timer)
  }, [intervalMs])
  
  return health
}
```

### 7.2 验证

1. Sidecar 启动时状态自动更新
2. Sidecar 崩溃时显示错误状态
