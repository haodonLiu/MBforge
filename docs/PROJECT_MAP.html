📊 项目概览

MBForge 是一款面向药物化学的桌面端知识库应用，核心定位类似 "Obsidian + Zotero" 的分子科学版本。 采用 PyQt6 构建 GUI，支持 PDF 深度解析、分子结构识别、向量语义检索与 LLM Agent 对话。

核心数据流说明

📄 流 1：PDF 入库（批量 / CLI） ▼

PDF 文件通过 FileProcessor 进入 PDFStrategy，最终由 PDFParserPipeline 处理：

  • 文本提取：PyMuPDF 逐页提取 → 分块 → 向量化存入 ChromaDB
  • 图片/VLM：提取内嵌图片（最多 20 张），可选 VLM 描述后追加到文本
  • LLM 摘要：生成 L0（一句话）和 L1（结构化概览），持久化到 JSON
  • 分子提取（文本路径）：正则提取 SMILES → RDKit 验证 → 直接入库
  • 分子提取（图像路径）：MolDetv2 检测 → MolScribe 识别 → ROI 文本关联 → pending 待确认

最终输出保存在 output/<doc_id>/ 目录下。

🔍 流 2：交互式 PDF 阅读 + 分子检测 ▼

用户打开 PDF 后，PDFViewer 采用虚拟滚动 + 线程池异步渲染：

  • 大文件（>100 页）自动启用 PDFSliceManager 分片
  • 点击「检测分子」触发 MolImagePipeline，在页面上叠加检测框
  • 用户可确认/拒绝/修正检测到的分子结构，批注持久化到 .mbforge/pdf_annotations/
  • 支持划词高亮、复制文本、区域提取分子
🤖 流 3：Agent 对话（ReAct 循环） ▼

ProjectAgent 执行最多 5 轮 ReAct 迭代：

  • L0：系统提示（永久）
  • L1：项目上下文 + MemoryManager 记忆注入（持久）
  • L2：工具执行结果（临时，不序列化）
  • L3：对话历史（可裁剪）

10 个工具覆盖：知识库搜索、文档查找、分子库查询、文档摘要读取等。

📤 流 4：Zotero 桥接（新增） ▼

Zotero 插件通过右键菜单收集条目元数据、PDF 路径和阅读批注，HTTP POST 到 localhost:8233：

  • MBForge Bridge 服务（aiohttp）接收后复制 PDF 到 zotero_imports/
  • 批注保存到 .mbforge/zotero_annotations/{key}.json
  • 自动加入项目索引，可选触发 PDFParserPipeline 解析

🔧 管线详解

PDF 解析主管线（PDFParserPipeline） ▼ 
fitz.open(pdf_path)
├── 文本提取 → split_text_chunks()
├── 图片提取（max 20, max 2MB）→ temp dir
├── VLM 图片描述（optional, max 5）→ append to text
├── DocumentSummarizer(llm)
│   ├── L0 prompt (~80 chars) → .mbforge/summaries/{doc_id}.json
│   └── L1 prompt (~1500 chars)
├── 分子提取（双路径）
│   ├── 图像路径（MolDetv2 + MolScribe）→ pending.json
│   └── 文本路径（Regex SMILES）→ mol_db [LIVE]
└── KnowledgeBase.index_document()
    └── Embedder.embed(chunks) [batch_size=64] → ChromaDB

已完成 该管线已完整实现，支持文本、图像双路径分子提取。

分子图像检测管线（MolImagePipeline） ▼ 
PIL Image / numpy array
├── MolDetv2DocDetector (960×960) → bbox list
├── MolDetv2GeneralDetector (640×640) → refinement
├── MolScribeRecognizer → SMILES + confidence
├── image_to_pdf_bbox() → PDF coordinate space
├── ROITextExtractor → caption/paragraph context
├── AssociationEngine → names / activities / cell lines
└── ExtractionResult(status="pending")

已完成 架构完整，需模型文件存放于 ~/.cache/mbforge/models/

UniParser 云端解析管线 ▼ 
ParserClient.parse_pdf()
├── trigger_file() → token
├── get_result() / get_formatted()
│   ├── textual=2 (OCRHighQuality)
│   ├── table=2 (OCRHighQuality)
│   ├── equation=2 (OCRHighQuality)
│   └── molecule=1 (OCRFast)
└── 输出：ParseResult → Markdown / SMILES / 表格

已完成 客户端封装完整，需外部 UniParser 服务。

文件处理策略管线（FileProcessor） ▼ 
process_file(entry, source_path, output_dir, **deps)
├── get_strategy(ext) → Strategy
├── strategy.extract() → ExtractedContent
├── strategy.index() → KB / mol_db
└── strategy.store() → output//index.json

支持策略：PDF / Markdown / Text / Molecule / DataTable / JSON

已完成 全部 6 种策略已实现。

Agent ReAct 管线（ProjectAgent） ▼ 
User input
├── LayeredContext.compose(L0-L3)
├── LLM.chat() → Tool call or Direct answer
├── If tool call:
│   ├── ToolExecutor.execute() → 10 tools
│   └── Result → L2 → next LLM call
└── TrajectoryTracker.record_step()

已完成 ReAct 核心完整，max_iterations=5 硬编码。
