# 003: tool_result 消息顺序错误

**日期：** 2026-05-21
**模块：** `src/mbforge/agent/context.py`
**状态：** 已解决

## 错误描述

与 001 相同的 API 错误，但根因不同：tool_result 消息被放入 `_tools` 层（ephemeral），该层在 `build_messages()` 中排在 `_history` 之前，导致消息顺序为：

```
system → project → [tool results] → [user, assistant+tool_use, tool results]
```

Anthropic API 要求 `assistant(tool_use)` 和 `user(tool_result)` 必须相邻。

## 解决方案

将 `add_tool_result()` 改为写入 `_history` 层而非 `_tools` 层，确保 tool_result 紧跟在 assistant 的 tool_use 消息之后。
