# Plan: Detection Cache Performance Optimization

## Goal
Eliminate redundant disk I/O and JSON deserialization during PDF page navigation by adding multi-level in-memory caching and adjacent-page prefetching.

## Problem Analysis

Current flow on every page switch:
1. Frontend clears `pageDetections` Map (line 294-310 of `usePdfViewer.ts`)
2. `handleDetectPage()` → `loadCachedDetections()` → Tauri IPC → `get_cached_page_detections`
3. Rust `DetectionCache::get()` reads `.schema_version` file from disk (every call!)
4. Rust `DetectionCache::get()` reads `page_NNNN.json` from disk
5. JSON deserialization → return to frontend

**Cost per page switch**: 2 synchronous file reads + 1 JSON parse (~1-5ms per page)
**Returning to a previously viewed page**: same cost, no caching

## Optimization Layers

### Layer 1: Schema Version Caching (Rust — `detection_cache.rs` domain)

**File**: `src-tauri/crates/mbforge-domain/src/document/detection_cache.rs`

Cache the `.schema_version` value per `base_dir` in a process-level static. The schema version is written once per project and changes only on software upgrades.

```rust
static SCHEMA_VERSION_CACHE: LazyLock<Mutex<HashMap<PathBuf, u32>>> = ...;

fn read_schema_version_cached(&self) -> Option<u32> {
    let dir = self.base_dir();
    // Check in-memory first, fall back to disk
}
```

**Impact**: Eliminates 1 disk read per `get()` call (the `.schema_version` file).

### Layer 2: In-Memory PageDetection LRU Cache (Rust — `detection_cache.rs` commands)

**File**: `src-tauri/crates/mbforge-app/src/commands/detection_cache.rs`

Add a process-level LRU cache for parsed `PageDetection` objects. Key: `(doc_id, page)`, Value: `(PageDetection, pdf_hash)`. Capacity: 100 pages.

```rust
static PAGE_CACHE: LazyLock<Mutex<HashMap<(String, usize), CacheEntry>>> = ...;
const PAGE_CACHE_CAPACITY: usize = 100;
```

- **On `get()` hit**: return from memory, skip disk entirely
- **On `get()` miss**: read from disk, insert into cache
- **On `put()`**: insert/update in cache + write to disk
- **On `clear_doc()`/`clear_all()`**: invalidate matching cache entries
- **On pdf_hash mismatch**: evict stale entry

**Impact**: Eliminates all disk I/O + JSON parse for repeated page visits (~0.01ms vs ~1-5ms).

### Layer 3: Adjacent Page Prefetching (Rust — `detection_cache.rs` commands)

**File**: `src-tauri/crates/mbforge-app/src/commands/detection_cache.rs`

After a successful `get_cached_page_detections` hit, spawn a background task to prefetch `page-1` and `page+1` into the LRU cache.

```rust
// After returning current page result:
tokio::spawn(async move {
    let _ = prefetch_adjacent_pages(&cache, &cache_key, page, &hash);
});
```

**Impact**: Next/prev page loads are instant (already in memory when user navigates).

### Layer 4: Frontend — Keep Detections Across Page Switches

**File**: `frontend/src/components/project/pdf/usePdfViewer.ts`

Change the `pageDetections` Map cleanup from "delete all non-current" to "keep a sliding window of 5 pages":

```typescript
// Before: delete all non-current
for (const key of next.keys()) {
    if (key !== currentPage) next.delete(key)
}

// After: keep ±2 pages around current
const WINDOW = 2
for (const key of next.keys()) {
    if (Math.abs(key - currentPage) > WINDOW) next.delete(key)
}
```

**Impact**: Navigating back 1-2 pages reuses the frontend Map without any IPC.

### Layer 5: Frontend — Prefetch Adjacent Pages via IPC

**File**: `frontend/src/components/project/pdf/usePdfViewer.ts`

After current page detection loads, fire-and-forget prefetch for ±1 pages:

```typescript
useEffect(() => {
    if (!pageDetections.has(currentPage)) return
    // Prefetch adjacent pages in background
    for (const p of [currentPage - 1, currentPage + 1]) {
        if (p >= 1 && p <= pdfPageCount && !pageDetections.has(p)) {
            void loadCachedDetections(p) // fire-and-forget
        }
    }
}, [currentPage, pageDetections, pdfPageCount])
```

**Impact**: Adjacent pages are pre-loaded into both backend LRU and frontend Map.
