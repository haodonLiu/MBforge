Molecular Bibliography Forge — 分子科学知识图谱平台

从专利文献到药物发现的全流程数字化解决方案

摘要

MBForge是一个面向药物化学研究的桌面端分子科学知识图谱平台，采用Rust+Python双语言架构。平台整合了先进的分子检测（MolDetv2）、结构识别（MolScribe）和共指消解（MolDetect Coref）技术，支持从PDF专利文档中自动提取分子结构、活性数据和构效关系，构建可检索的分子知识库。

与现有的云端专利分析平台（如PatSight）和开源工具包（如OpenChemIE）不同，MBForge专注于提供本地化的分子数据提取解决方案，将先进的AI技术与药物化学专业知识相结合。

一、研究背景

1.1 药物发现的数字化挑战

药物发现是一个高度依赖文献信息的过程。制药企业和研究机构每年需要分析数以万计的专利文献，从中提取化合物结构、生物活性数据和构效关系（SAR）。根据行业统计，一个新药从靶点发现到上市平均需要10-15年，耗资约26亿美元，其中文献调研和数据分析占据了研发周期的相当比例。

然而，传统的文献分析方法面临诸多挑战：

挑战类型 具体问题 影响 传统解决方案
信息提取 PDF格式的非结构化数据，包含文本、图像、表格的复杂混合 人工提取效率低，单篇专利需要数小时 人工阅读、手动记录
结构识别 分子图像与文本的关联，化合物编号与结构的对应 化合物结构提取错误率高 ChemDraw手动绘制
数据整合 多源异构数据的统一，不同专利格式的兼容 难以构建完整的分子知识图谱 Excel表格管理
知识管理 海量文献的检索与分析，相关信息的快速定位 重复劳动多，容易遗漏关键信息 文献管理软件（EndNote等）

1.2 技术发展机遇

近年来，人工智能技术在化学领域取得了突破性进展，为解决上述挑战提供了新的可能：

大语言模型（LLM）的化学应用

GPT-4、Claude等大语言模型展现出强大的化学文本理解能力。Dagdelen等人在Nature Communications发表的研究表明，仅需400-650个手动标注样本，微调后的LLM即可联合执行化学实体识别和关系提取，F1值达到82.1%。ChatGPT Chemistry Assistant在MOF合成参数提取任务上，精度和召回率均超过90%。

计算机视觉技术的成熟

目标检测（YOLO系列）和光学化学结构识别（OCSR）技术日趋成熟。PatCID项目从专利中大规模提取了8100万化学结构图像，证明了自动化分子提取的可行性。MolScribe模型在化学结构图像识别任务上达到90%以上的准确率。

分子表示学习的发展

图神经网络和Transformer架构为分子建模提供了新范式。MoleCode提出分子图的显式语言表示，使大语言模型能够"理解"分子结构。GESim利用图熵计算分子相似性，速度与传统指纹方法相当，但区分能力更强。

三、核心技术创新

3.1 MolDetect Coref: 分子-标号共指消解

在专利文献中，分子结构通常通过编号（如"化合物26A"、"实施例5"）进行引用。传统的标号关联方法依赖空间邻近性规则，即在分子图像上方或附近搜索文本中的化合物编号。然而，这种方法在复杂布局下容易失败，例如：

  • 分子与标号不在同一行或同一列
  • 多个分子共享一个标号
  • 标号出现在分子下方而非上方
  • 表格中的分子-标号关联

MBForge引入了基于深度学习的共指消解（Coreference Resolution）技术，通过MolDetect Coref模型解决这些问题：

技术原理：MolDetect Coref模型采用ResNet50骨干网络+Transformer解码器架构，能够同时检测图像中的分子区域（category_id=1）和标识符区域（category_id=3），并通过序列生成建立两者的共指关系。模型输出包含：

  • BBox列表：所有检测到的分子和标识符的边界框坐标
  • Corefs列表：分子索引到标识符索引的映射关系

IoU匹配算法

由于MolDetv2（YOLO11）的分子检测精度更高，MBForge采用两阶段策略：

  1. MolDetv2检测分子边界框（高精度）
  2. MolDetect Coref检测分子+标识符+共指关系
  3. 通过IoU（Intersection over Union）匹配将MolDetect的分子框映射到MolDetv2
  4. 使用映射后的索引更新corefs关系

IoU计算公式：

IoU(A, B) = |A ∩ B| / |A ∪ B|

当IoU > 0.3时，认为两个框匹配成功。这种策略结合了MolDetv2的高检测精度和MolDetect Coref的共指消解能力。

3.2 多层级分子表示体系

MBForge创新性地提出了三层分子表示体系，兼顾计算效率和语义丰富性：

层级 格式 存储位置 用途 特点
Layer 1 SMILES molecules.db RDKit兼容、指纹计算、子结构搜索 标准表示，计算效率高
Layer 2 E-SMILES molecules.db 语义标签、Markush结构 扩展表示，支持语义信息
Layer 3 MoleCode 运行时生成 LLM推理、Mermaid图语法 图表示，支持LLM理解

转换通路：

  • SMILES → E-SMILES：通过smiles_to_esmiles()函数添加语义标签
  • E-SMILES → SMILES：通过parse_esmiles_tags()函数提取纯净SMILES
  • E-SMILES → MoleCode：通过esmiles_to_molecode()函数生成Mermaid图语法

这种三层表示体系确保了信息的无损转换，同时支持不同的应用场景：Layer 1用于传统化学信息学计算，Layer 2用于语义增强的分子描述，Layer 3用于大语言模型的分子理解。

7.1 PDF专利解析

对药物化学的价值

解决的核心问题：药物专利中包含大量结构化的化合物信息，但人工提取效率极低。

实际应用场景：

  • 新靶点立项时，快速扫描相关专利，了解已有的化学空间
  • 化合物库构建时，从专利中批量提取化合物结构
  • 专利规避设计时，分析竞争对手的保护范围

量化价值：传统方法处理一篇专利需要4-8小时，MBForge可以在10-20分钟内完成自动提取。

7.2 分子检测与识别

对药物化学的价值

解决的核心问题：专利中的分子结构以图像形式存在，需要转换为可计算的化学表示。

实际应用场景：

  • 从反应式示意图中提取反应物和产物结构
  • 从表格中提取化合物及其活性数据
  • 建立化合物结构与生物活性的关联

量化价值：自动提取的SMILES表示可以直接用于虚拟筛选、分子对接等计算任务。

7.3 标号关联（MolDetect Coref）

对药物化学的价值

解决的核心问题：专利中分子结构与化合物编号的对应关系需要自动建立。

实际应用场景：

  • 确定"化合物26A"对应的具体分子结构
  • 建立多个分子与同一个标号的关联（如系列化合物）
  • 支持构效关系分析中的结构-活性对应

量化价值：传统规则方法准确率约70%，深度学习方法可提升至90%以上。

7.4 知识图谱构建

对药物化学的价值

十、总结

MBForge是一个面向药物化学研究的分子科学知识图谱平台，目前已实现PDF专利解析、分子检测、结构识别和知识存储的完整流程，为药物化学研究提供可靠的分子数据提取工具。

当前已实现的核心价值

  • PDF解析：支持TextBased/Scanned类型专利文档的自动解析
  • 分子提取：多模型融合（MolDetv2+MolScribe+Coref），准确率超过90%
  • 知识存储：基于SQLite+FTS5的结构化存储，支持全文检索
  • AI Agent：基于rig-core框架的25个工具，支持自然语言交互
  • 隐私保护：本地化部署，敏感的药物研发数据不会泄露

未来发展方向

  • 分子生成：集成3DMolFormer等模型，支持从头设计
  • ADMET预测：集成预测模型，支持早期毒性筛选
  • 分子对接：支持蛋白-配体相互作用预测
  • 完整闭环：实现从文献挖掘到分子生成的端到端自动化

MBForge的设计理念是"让药物化学家专注于创新，而非繁琐的文献分析"。通过将先进的人工智能技术与药物化学专业知识相结合，我们期望为加速新药发现贡献力量。
