# 001: Anthropic API tool_use/tool_result ID 不匹配

**日期：** 2026-05-21
**模块：** `src/mbforge/models/anthropic_llm.py`, `src/mbforge/agent/agent.py`
**状态：** 已解决

## 错误描述

```
anthropic.BadRequestError: Error code: 400 - {'type': 'error', 'error': {
  'type': 'invalid_request_error',
  'message': "invalid params, tool result's tool id(call_xxx) not found (2013)"
}}
```

## 根因分析

`AnthropicLLM._convert_messages()` 在转换 tool result 消息时，故意省略了 `tool_use_id` 字段。注释说明是因为"MiniMax Anthropic 兼容接口不支持外部传入的工具调用 ID"，但实际上 MiniMax 的 Anthropic 兼容 API 需要这个字段来匹配 tool_use 和 tool_result。

同时，`add_assistant_message()` 只存储文本内容，丢失了 `tool_use` 块，导致第二轮请求时历史中没有对应的 `tool_use`。

## 解决方案

1. `base.py` — Message 新增 `tool_calls` 字段
2. `context.py` — `add_assistant_message` 支持传入 `tool_calls`
3. `agent.py` — 两处 `add_assistant_message` 调用传入 `tool_calls`
4. `anthropic_llm.py` — `_convert_messages` 正确生成带 `tool_use_id` 的 `tool_result` 块，以及带 `tool_use` 内容块的 assistant 消息
